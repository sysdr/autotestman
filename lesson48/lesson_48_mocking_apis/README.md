# Lesson 48: Mocking APIs

## Quick Start
```bash
pip install requests responses pytest pytest-html
pytest tests/ -v --html=report.html --self-contained-html
python utils/visualizer.py
```

## File Structure
```
lesson_48_mocking_apis/
├── reports/
│   └── report_visualizer.html   # Generated by utils/visualizer.py
├── tests/
│   ├── conftest.py              # Shared pytest fixtures (mock registration)
│   └── test_graphql_mock.py    # All test cases
└── utils/
    └── visualizer.py            # HTML request/response visualizer
```

## Key Concepts
- `responses.RequestsMock` — intercepts `requests` at the adapter layer
- `yield` in fixtures — clean setup/teardown, zero leaked state
- GraphQL POST structure — `{"query": "...", "variables": {...}}`
- `len(mock.calls) == 1` — proves no double-call bugs
